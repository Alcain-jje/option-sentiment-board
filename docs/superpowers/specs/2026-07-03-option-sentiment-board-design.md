# 옵션 심리 보드 (Option Sentiment Board) — 설계 문서

- 작성일: 2026-07-03
- 상태: 사용자 승인 대기
- 용도: 개인 학습/토이 프로젝트 (공개 서비스 아님)

## 1. 개요

미국 대장주의 옵션 시장 데이터(풋/콜 거래량·미결제약정)를 매일 수집해,
초보자도 "이 종목에 상승 베팅이 많은지 하락 베팅이 많은지"를 한눈에 파악할 수 있는
정적 웹사이트. 전문 용어(풋/콜, OI) 대신 **0~100 심리 점수 + 한글 한 줄 해석**으로 압축한다.

### 목표
- 종목별 시장 심리(강세/약세 베팅 쏠림)를 5초 안에 읽히는 UI로 제공
- 서버·비용 0원: GitHub Actions(수집) + GitHub Pages(호스팅)만 사용
- 매일 아침(KST) 자동 갱신, 사람 개입 불필요

### 비목표 (v1에서 하지 않는 것)
- 실시간/장중 데이터 (EOD 하루 1회로 충분)
- 한국 종목 (Phase 2로 연기 — 공매도 잔고 기반 별도 설계)
- 사용자 계정, 알림, 매매 신호 제공 (투자 조언 아님 — 면책 문구 표기)

## 2. 데이터 소스

| 데이터 | 소스 | 방식 | 비고 |
|---|---|---|---|
| 종목별 옵션체인 (행사가별 콜/풋 volume·OI·IV) | Yahoo Finance | `yfinance` Python 라이브러리 | API키 불필요, 15분 지연(EOD엔 무관) |
| 주가(현재가·전일종가) | Yahoo Finance | `yfinance` | 카드에 가격·등락 표시용 |
| 시장 전체 풋/콜 비율 | 종목 50개 합산으로 자체 계산 | — | CBOE 공식 지표와 다름을 UI에 명시 |

### 데이터 신선도 주의사항
- 거래량(volume): 당일 장마감 값
- 미결제약정(OI): OCC 정산 특성상 **전일 마감 기준**일 수 있음 → UI에 "기준 시점" 명시
- yfinance는 비공식 API라 간헐 실패 가능 → 재시도·부분실패 허용 설계(§6)

## 3. 대상 종목 (52개)

시총 상위 + 옵션 유동성이 높은 종목. 수집 시 옵션 총거래량이 임계치(1,000계약) 미만이면
해당일 점수 계산을 건너뛰고 "데이터 부족" 표시.

```
빅테크/반도체: AAPL MSFT NVDA AMZN GOOGL META TSLA AVGO AMD QCOM
              INTC MU TXN ARM SMCI ORCL CRM ADBE NFLX CSCO
금융:          JPM BAC GS V MA BRK-B
헬스케어:      LLY UNH JNJ ABBV MRK PFE
소비/산업:     WMT COST HD PG KO PEP DIS NKE MCD BA CAT GE XOM CVX
고변동/밈:     PLTR COIN GME MSTR HOOD UBER
```

(52개, 티커 목록은 `collector/tickers.py` 단일 파일에서 관리 — 추가/제거 용이)

## 4. 심리 점수 알고리즘

### 4.1 원재료
종목별로 **만기 60일 이내의 모든 옵션체인**을 합산:
- `call_vol`, `put_vol` (당일 거래량 합)
- `call_oi`, `put_oi` (미결제약정 합)
- 행사가별 콜/풋 OI 분포 (상세 화면 "베팅 지도"용, 현재가 ±20% 범위)

### 4.2 점수 계산 (0~100, 높을수록 강세)
1. `pc_ratio = put_vol / call_vol` (당일)
2. **정규화 모드 (기본)**: 해당 종목의 최근 60거래일 `pc_ratio` 분포에서 당일 값의 백분위 `p`를 구해
   `score = round(100 × (1 − p))` — 낮은 풋콜비율(=콜 우세)일수록 높은 점수
3. **콜드스타트 모드**: 이력이 20거래일 미만이면 절대값 구간 매핑 사용
   - `pc_ratio` 0.4 이하 → 90, 0.4~1.6 구간 선형 매핑 (0.7→70, 1.0→50, 1.3→30), 1.6 이상 → 10
   - 이력이 쌓이면 자동으로 정규화 모드 전환 (JSON에 `scoreMode` 필드로 표기)
4. 라벨 매핑: 80+ 매우 강세 / 60~79 강세 / 40~59 중립 / 20~39 약세 / ~19 매우 약세

### 4.3 한 줄 해석 생성 (규칙 기반, LLM 불사용)
- 콜/풋 배율: `call_vol/put_vol` 또는 역수를 "상승 베팅이 하락의 N.N배" 형태로
- 60일 백분위가 상·하위 10% 이내면 "평소보다 훨씬/뚜렷한" 수식어 추가
- 중립(40~59)이면 "상승·하락 베팅 비슷 — 관망 분위기"

## 5. 아키텍처

```
option-sentiment-board/  (GitHub public repo)
├─ collector/                  # Python 수집기
│  ├─ tickers.py               # 종목 50개 정의
│  ├─ collect.py               # 메인: 수집→점수계산→JSON 출력
│  ├─ scoring.py               # §4 알고리즘 (순수함수, 단위테스트 대상)
│  └─ tests/
├─ site/                       # 정적 프론트 = GitHub Pages 서빙 루트
│  ├─ index.html               # 메인 보드
│  ├─ stock.html               # 상세 화면 (?t=NVDA)
│  ├─ app.js / style.css
│  └─ data/                    # Actions가 커밋하는 산출물 (프론트가 상대경로 fetch)
│     ├─ latest.json           # 전 종목 요약 (메인 보드 1회 fetch)
│     ├─ history/{TICKER}.json # 일별 이력 (rolling 120일)
│     └─ detail/{TICKER}.json  # 행사가별 OI 분포 + 30일 점수 흐름
└─ .github/workflows/collect.yml
```

- **수집 스케줄**: cron `10 21 * * 2-6` (UTC) = 화~토 KST 아침 6:10, 미 장마감(서머타임 20:00 UTC) 후
- **배포**: GitHub Pages — `main` 브랜치의 `/site` 디렉토리 서빙(actions/deploy-pages). 데이터 갱신은 Actions의 `site/data/*.json` 커밋으로 완결 (프론트는 상대경로 fetch만)
- **프론트 스택**: 빌드 도구 없는 vanilla HTML/CSS/JS. 차트는 CSS 막대(베팅 지도·30일 흐름 모두 div 막대로 충분) — 외부 라이브러리 0개

### latest.json 스키마 (요약)

## 6. 오류 처리

- 종목 단위 재시도 2회(지수 백오프) 후 실패 시 해당 종목만 skip, 직전 데이터 유지 + `stale: true`
- 실패 종목이 30% 초과하면 데이터 커밋 없이 워크플로 실패 처리 (GitHub이 실패 메일 발송)
- 프론트: `stale` 종목은 카드에 "어제 데이터" 배지, `updatedAt`이 48시간 이상 과거면 상단 경고 배너
- 주가 0/음수, `call_vol=0`(분모 0) 등 이상값은 scoring 단계에서 방어 (데이터 부족 처리)

## 7. UI 설계

목업(대화에서 승인됨) 기준. 사용자 피드백 반영: **여백을 넉넉하게** — 콘텐츠 최대폭 1080px
중앙 정렬, 카드 한 줄 2~3개, 카드 내부 패딩·행간 목업 대비 1.3배 수준.

- **메인 보드**: 상단 시장 전체 게이지 → 종목 카드 그리드(점수, 게이지 바, 한 줄 해석, 주가·등락). 정렬 토글: 강세순/약세순/시총순. 색 관례: 빨강=강세, 파랑=약세 (국내 증권앱 관례)
- **상세 화면**: 심리 점수 + 서술형 요약 / 베팅 지도(행사가별 콜·풋 OI 나비 차트) / 30일 심리 흐름 막대 / 초보자 안내 박스("옵션 심리는 예언이 아니라 현재 돈의 쏠림")
- 다크모드: v1 제외. 모바일: 카드 1열 폴백만 지원
- 하단 면책: "본 사이트는 학습용이며 투자 조언이 아닙니다" + 데이터 출처·기준시점

## 8. 테스트 전략

- `scoring.py` 순수함수 단위테스트(pytest): 콜드스타트 경계(20일), 백분위 경계, 분모 0, 극단값
- 수집기: yfinance 응답을 fixture로 목킹한 통합테스트 (실네트워크 테스트는 CI에서 제외)
- 프론트: latest.json fixture로 로컬 서빙 후 수동 확인 + Claude Preview 브라우저 QA
- 파이프라인 검증: 저장소 push 후 workflow_dispatch로 수동 1회 실행 → Pages 반영 확인
