### collect.py 변경 (TDD: 테스트 먼저 갱신)

1. 루프 언패킹: `for ticker, name in pairs` → `for ticker, name, sector in pairs`
2. ok 스톡 dict에 3개 필드 추가:
   - `"sector": sector`
   - `"spark": [h["score"] for h in history[-10:]]` (최근 ≤10일 점수, 오늘 포함)
   - `"scoreChg": score - history[-2]["score"] if len(history) >= 2 else None`
3. insufficient 스톡 dict에도 `"sector": sector` 추가 (spark/scoreChg는 없음)
4. stale 재사용 경로는 변경 없음 (직전 entry에 이미 sector/spark 포함되어 복사됨)

### test_collect.py 갱신

- 모든 fixture pair를 3-튜플로: `("NVDA", "엔비디아", "반도체")`, `[(s, s, "테스트") for s in ...]`
- happy path에 추가 assert:
  ```python

  assert s["spark"] == [80]
  assert s["scoreChg"] is None
  ```
- 신규 테스트 — 이틀치 이력 시 spark 2개·scoreChg 계산:
  ```python
  def test_spark_and_score_chg_accumulate(tmp_path, monkeypatch):
      monkeypatch.setattr(collect, "fetch_ticker", _fake_fetch_ok)
      pairs = [("NVDA", "엔비디아", "반도체")]
      collect.run(pairs, tmp_path)
      # 전일 이력을 인위 조작(오늘보다 하루 전 날짜, 점수 70)
      import json as _json
      hp = tmp_path / "history" / "NVDA.json"
      hist = _json.loads(hp.read_text(encoding="utf-8"))
      hist.insert(0, {"date": "2000-01-01", "pcRatio": 0.8, "score": 70})
      hp.write_text(_json.dumps(hist), encoding="utf-8")
      collect.run(pairs, tmp_path)
      latest = _json.loads((tmp_path / "latest.json").read_text(encoding="utf-8"))
      s = latest["stocks"][0]
      assert s["spark"] == [70, 80]
      assert s["scoreChg"] == 10
  ```

승인된 목업 기준. 데이터 계약: stocks[]에 `sector`(항상), `spark`(ok만, ≤10개 배열), `scoreChg`(ok만, int|null) 추가됨.

### 레이아웃
- `.wrap` max-width **1320px** (기존 1080), 패딩 유지
- 카드 그리드: `repeat(auto-fill, minmax(270px, 1fr))` → 1320px에서 4열, 모바일 1열 유지

### 상단 요약 밴드 (기존 .market 섹션 대체)
- grid `2fr 1fr 1fr 1fr` (모바일: 2×2 wrap)
- 칸1(게이지): "시장 전체 분위기" + 점수/라벨 + 기존 게이지 바 (marketScore null이면 이 칸 전체 숨김)
- 칸2: "강세 종목" — score≥60 개수 / 전체 (빨강 계열 숫자)
- 칸3: "약세 종목" — score≤39 개수 / 전체 (파랑 계열)
- 칸4: "오늘 최다 쏠림" — |score−50| 최대 종목의 티커+점수 (클릭 시 상세로)

### 섹터 필터 칩 (요약 밴드와 정렬 토글 사이)
- "전체" + 데이터에서 뽑은 섹터들(ok 종목 수 기준 내림차순), 각 칩에 섹터 평균점수 표시(반올림, 점수 색상 규칙 동일)
- 클릭 시 해당 섹터만 필터 (활성 칩 강조: 검정 배경). 정렬 토글과 독립 조합
- 필터 상태에서 요약 밴드는 전체 기준 유지(단순성)

### 카드 v2
- 1행: 티커+한글명(좌) / 점수 22px(우)
- 점수 아래 scoreChg: null 아니고 0 아니면 `▲ 어제보다 +N`(빨강)/`▼ 어제보다 -N`(파랑), 0이면 "어제와 같음"(회색), null이면 미표시
- 티커 아래 섹터 뱃지(회색 미니 pill) + stale 뱃지(기존 로직 유지)
- 스파크: 높이 26px, `spark` 배열을 flex 막대로(높이 = max(score,8)%, 색은 각 점수의 barColor). 배열 길이 1이어도 렌더
- 가격·등락 + ratioText 기존 유지
- insufficient 카드: 기존 문구 유지 + 섹터 뱃지 표시
- 기존 게이지 바(.bar)는 스파크와 중복이므로 **제거** (점수 색상+스파크가 대체)

### 구현 방식
app.js는 상태 객체 `{sort: "desc", sector: null}` 하나로 관리, 칩/토글 클릭 시 상태 갱신 → render(). 나머지 구조(fetch, cardHtml, 에러 fallback)는 기존 패턴 유지. 시총순은 여전히 범위 외.

검증: 로컬 http.server + (컨트롤러가 브라우저 QA 수행). 커밋: `feat: 보드 리디자인(요약밴드·섹터칩·스파크·4열)`
